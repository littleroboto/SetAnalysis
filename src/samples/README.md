# Samples

file is bundled into the static build via Vite's `?raw` import, so it travels
with `dist/` and works on the deployed Pages site without a fetch.

## Regenerating synthetic YAML

```bash
cd viz/upset-tool
npm run gen:uk-passb   # rewrites src/samples/synthetic-uk-passb.yaml
npm run gen:de-passb   # rewrites src/samples/synthetic-de-passb.yaml
npm run gen:pl-passb   # rewrites src/samples/synthetic-pl-passb.yaml
npm run gen:nl-passb   # rewrites src/samples/synthetic-nl-passb.yaml
npm run gen:uk-params-passb  # rewrites src/samples/synthetic-uk-parameters-passb.yaml
npm run gen:pl-params-passb  # rewrites src/samples/synthetic-pl-parameters-passb.yaml
npm run gen:uk-dual-passb
npm run gen:de-dual-passb
npm run gen:pl-dual-passb
npm run gen:nl-dual-passb
```

Generators are deterministic for a fixed `SEED` constant. **Do not edit the
output YAML by hand** &mdash; tweak the script (archetypes, counts, seed) and
regenerate, so the file stays reproducible.

## Adding another market sample

1. Drop the YAML under `src/samples/` (the path is what shows up in the
   `meta.source` field; keep them aligned).
2. Register it in `src/main.ts` &rarr; `SAMPLES` array.
3. Document the pedagogical purpose in the table above: filename, one-line
   description, and (per the prompt) the marginal interpretation a naive
   reader would reach versus the bundle interpretation the chart reveals.
4. If the sample is generated, commit the generator script alongside it
   under `_generators/` and reference the seed in `meta.notes`. The
   generator runs under `_generators/tsconfig.json`, which adds Node types
   on top of the strict app config &mdash; so the browser bundle stays free
   of `node:*` imports.
